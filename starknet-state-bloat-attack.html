Key Findings

  • Perfect linear scaling for storage: The final storage grows exactly proportionally to the number of attackers (scaling factor: 1.0)
  • L2 gas usage scales slightly sub-linearly with the number of attackers (scaling factor: 0.95), showing small efficiency gains as the attack scales
  • L1 data gas shows significant sub-linear scaling (scaling factor: 0.86), indicating better efficiency at scale
  • Per-attacker efficiency improves as the number of attackers increases, with the most significant improvement seen when scaling from 10 to 20 attackers
  • Each attacker requires exactly 20,000 ETH as minimal stake amount for all test runs
